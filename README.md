# X/Twitter Following Manager

英文商店名：**X/Twitter Following Manager - Instantly Show Follower Counts & Export Lists**

中文商店名：**X/Twitter 关注管理器 - 直接显示粉丝数，管理并导出名单**

一个用于清理和浏览 X/Twitter 关注名单的小扩展：在 Feed 流和用户列表的昵称旁直接显示粉丝数和关注数据，减少反复悬停头像的操作，并支持导入、导出和顺序更新账号数据。

## 安装

1. 打开 Chrome 或 Edge 的扩展管理页面。
2. 开启“开发者模式”。
3. 选择“加载已解压的扩展程序”。
4. 选择这个目录：`/Users/liuqi/Desktop/code/codex/x-follow-stats-extension`。
5. 打开或刷新 X 的关注列表页面，例如 `https://x.com/你的用户名/following`。

## 使用

刷新 X 页面后，扩展会在博主昵称旁显示类似：

```text
388 正在关注 · 2.6万 关注者
```

如果已经打开了页面再安装扩展，建议刷新一次。滚动页面时，新出现的条目会继续补齐。

扩展弹窗提供显示和更新开关：

- Feed 流：控制主页、列表、个人主页等信息流里的推文作者。
- 用户列表：控制关注页、粉丝页、搜索用户结果、右侧用户模块等用户条目。
- 锁定账号列表：只更新当前已经捕获或导入的账号，不把浏览时新看到的账号加入管理页。

界面默认跟随浏览器 UI 语言自动使用中文或英文，也可以在弹窗右上角手动切换。

通知页不会显示统计标签，避免把点赞、回复、聚合通知里的头像误判成用户列表。

弹窗里也可以手动“重新扫描本页”或“清除缓存”。

## 用户管理页

弹窗里的“打开用户管理页”会进入独立页面。这里会汇总浏览过程中捕获到的用户，并支持：

- 搜索昵称、ID、简介、网站和已捕获推文。
- 按来源筛选：Feed 流、用户列表、导入。
- 控制是否显示评论列表来源；评论列表不会再被算作 Feed 流，同一用户如果也出现在发布推文或用户列表中，隐藏评论列表来源时仍会保留。
- 控制“评论优先”，用于同一用户同时有发布推文和评论时选择显示哪一类内容；它只影响已浏览的最新推文，不影响用户来源筛选。
- 勾选用户后导出 CSV 或 Markdown。
- 从扩展导出的 CSV 或 Markdown 重新导入用户。
- 自定义导入时，最低需要保留账号主页链接，例如 `https://x.com/example`；扩展会从链接里提取 handle。
- 批量打开已选用户主页，便于重新浏览并刷新离线数据。
- 用刷新图标顺序更新已选用户：扩展会打开一个专用标签页，逐个访问主页、轻微滚动并等待统计数据写入，避免一次性打开大量标签页；切回管理页查看进度不会中断，关闭刷新标签页才会停止。
- `选中缺失结果` 会选中当前结果里正在关注数或关注者数不完整的用户；`反向选择` 会反转当前结果的选中状态。
- `锁定账号列表`：适合批量补齐缺失数据，只更新已经在管理页里的账号，不把新看到的账号加入缓存。
- 查看从 Feed 流捕获到的最新内容，并标注“原创 / 转推 / 评论”；用户列表或导入来源但未在 Feed 出现的账号会说明对应原因。
- 导入文件里没有推文类型时，不会默认标成“原创”。
- 导入来的推文内容在管理页标为“导入”；之后重新浏览 X 捕获到的新内容，会按实际内容显示“原创 / 转推 / 评论”。

CSV 默认包含这些列：昵称、handle、用户 ID、主页链接、正在关注数、关注者数、来源、简介、网站、首次看到时间、最近看到时间、最新推文类型、最新捕获推文、推文链接、捕获推文数量。

## 数据来源

扩展优先读取 X 页面自己已经请求到的用户列表数据，不需要登录信息、API key 或第三方服务。对没有捕获到的数据，会尝试读取头像悬停卡片。缓存只保存在浏览器本地。

## 已知限制

- X 的页面结构和内部接口可能变化，变化后需要更新选择器或解析逻辑。
- 刚安装后，如果当前页面早已加载完数据，可能需要刷新页面才能一次性显示更多数字。
- 私密、异常或被限制的账号可能无法显示完整数据。
